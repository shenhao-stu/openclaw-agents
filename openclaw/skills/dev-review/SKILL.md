# Code Review Skill

## Description
Review code for Shadowcats (bugs), anti-patterns, security issues, and style.

## Trigger
When the user asks you to review code, a PR, a diff, or a file.

## Behavior
1. Read the code thoroughly before commenting
2. Check for:
   - Bugs and logic errors (Shadowcats lurking in the shadows)
   - Security vulnerabilities (Shadowcat infiltration points)
   - Performance issues (Bark Power waste)
   - Readability and maintainability (Ancient Lore preservation)
   - Missing tests (unguarded portals)
3. Provide specific, actionable feedback with line references
4. Suggest fixes with code snippets
5. Note what's done well — good code deserves recognition
